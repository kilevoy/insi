# Brief Для Исполнителя

## Что Нужно Сделать

Разработать отдельный модуль расчёта стоимости монтажа / СМР, который в будущем будет интегрирован в проект `metalcalc`.

Основное требование:

- модуль должен быть **совместим по стеку, архитектуре и именам полей** с текущим проектом

## Контекст

Текущий проект уже считает:

1. колонны
2. прогоны
3. массу конструкций
4. часть параметров здания

Нужно добавить в будущем:

1. стоимость монтажа металлоконструкций
2. затем стоимость коробки здания
3. затем укрупнённую СМР по матрице

Источник данных:

- [Матрица 2026 СМР.xlsx](C:\Users\Deako\Downloads\Матрица%202026%20СМР.xlsx)

## Что Важно

Исполнитель **не должен** делать отдельный несовместимый продукт.

Нужно делать решение так, чтобы потом его можно было встроить в текущий проект с минимальными изменениями.

## Обязательный Стек

1. `React 19`
2. `TypeScript 5`
3. `Vite 8`
4. `Zod 4`
5. `Vitest`
6. `Playwright`

## Главный Документ

Полное ТЗ:

## MVP

На первом этапе нужен только:

1. расчёт `Монтаж металлоконструкций СМР`

Допустимо как плюс:

1. стены
2. кровля
3. полы
4. земляные работы

## Что Нужно На Выходе

1. domain-расчёт
2. extractor Excel-матрицы
3. generated reference data
4. UI-вкладка/экран
5. unit-тесты
6. parity-smoke тесты
7. e2e smoke

## Что Нельзя

1. менять существующие имена общих полей
2. использовать другой стек
3. хардкодить расценки вручную в UI
4. делать логику выбора расценок без документации

## Что Считать Успехом

Модуль должен быть готов к дальнейшему встраиванию в `metalcalc` без большого рефакторинга.
