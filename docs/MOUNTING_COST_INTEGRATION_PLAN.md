# План Подвязки Расчёта Монтажа К Калькулятору

## Цель

Подвязать укрупнённый расчёт стоимости СМР/монтажа к текущему калькулятору здания на основе матрицы:

- [Матрица 2026 СМР.xlsx](C:\Users\Deako\Downloads\Матрица%202026%20СМР.xlsx)

Так, чтобы пользователь мог получить:

1. стоимость монтажа металлокаркаса
2. стоимость коробки здания
3. в будущем укрупнённую смету по основным разделам

## Что Представляет Собой Исходный Excel

Файл `Матрица 2026 СМР.xlsx` — это не полноценный калькулятор, а укрупнённая сметная матрица с:

- видами работ
- единицами измерения
- количеством
- сроками
- ценой за единицу
- итоговой стоимостью
- региональными коэффициентами

Ключевые разделы:

1. Комплексные изыскания / геология
2. Проектные работы
3. Земляные работы
4. Бетонные работы
5. Полы
6. Металлоконструкции СМР
7. Стены и перегородки
8. Крыша / кровля
9. Окна / двери / ворота
10. Водоснабжение / канализация
11. Теплоснабжение
12. Электромонтаж
13. Дополнительные работы

Ниже есть региональные коэффициенты:

- Челябинская область
- Свердловская / Тюменская области
- ХМАО
- ЯНАО
- Поволжский ФО
- Южный ФО

## Что Уже Можно Брать Из Текущего Калькулятора

Из существующего расчёта уже можно получать или выводить:

1. площадь здания
2. пролет
3. длину
4. высоту
5. тип кровли
6. уклон кровли
7. массу колонн
8. массу прогонов
9. суммарную массу металлоконструкций
10. наличие снегозадержания / ограждения кровли

Это уже даёт основу для расчёта некоторых разделов СМР.

## Что Можно Подвязать На Первом Этапе

### MVP 1. Монтаж металлоконструкций

Самый реалистичный и полезный первый этап.

Основание из матрицы:

- раздел `6. МЕТАЛЛОКОНСТРУКЦИИ СМР`
- единица: `тн`
- база: `вес конструкций`

Что нужно:

1. суммарная масса колонн
2. суммарная масса прогонов
3. флаг типа каркаса:
   - `ЛСТК`
   - `сортовой / обычные металлоконструкции`
4. правило выбора ценовой строки из 4 вариантов

Результат:

- стоимость монтажа металлоконструкций

### MVP 2. Коробка здания

Следующий полезный уровень.

Можно подтянуть:

1. Земляные работы
   - база: `площадь здания`
2. Бетонные работы
   - база: `площадь здания`, но нужен тип фундамента
3. Полы
   - база: `площадь здания`
4. Стены
   - база: `площадь стен`
5. Кровля
   - база: `площадь кровли`
6. Кровельные аксессуары
   - снегозадержатели
   - ограждения
   - водосточка

Результат:

- укрупнённая стоимость коробки здания

## Что Пока Не Хватает Для Полной Подвязки

Для полного расчёта по всей матрице текущих входных данных недостаточно.

Не хватает как минимум:

1. типа фундамента
2. толщины/типа пола
3. площади стен в явном виде
4. площади кровли в явном виде
5. длины карнизов / периметра кровли
6. длины водостоков / желобов / труб
7. количества и площади:
   - окон
   - дверей
   - ворот
8. флагов по инженерии:
   - водоснабжение / канализация
   - отопление
   - электрика
9. региона для коэффициента накладных расходов
10. правила выбора уровня цены из нескольких строк матрицы

## Главная Методическая Проблема

Матрица не содержит явного правила:

- какую из 4 строк расценки выбирать для конкретного здания

Например:

- для металлоконструкций есть 4 цены
- для стен есть 4 цены
- для кровли есть 4 цены

Но в файле нет формализованной логики выбора по:

- площади
- массе
- этажности
- сложности
- региону
- типу объекта

Значит для внедрения нужно отдельно зафиксировать бизнес-правила выбора строки.

## Рекомендуемая Архитектура Интеграции

### 1. Новый доменный модуль

Добавить отдельный слой:

- `src/domain/mounting-cost/model/*`

Предлагаемые файлы:

- `mounting-cost-input.ts`
- `mounting-cost-output.ts`
- `mounting-cost-reference.generated.ts`
- `calculate-mounting-cost.ts`
- `mounting-cost-mapper.ts`

### 2. Новый reference generator

Считать матрицу в отдельный generated-модуль.

Нужен скрипт:

- `scripts/extract_mounting_cost_reference.py`

Что хранить:

1. список разделов
2. список строк
3. unit
4. basis
5. unit price
6. region coefficients

### 3. Отдельный mapping из unified input

Нужен mapper:

- из параметров здания
- из результатов колонн
- из результатов прогонов

в укрупнённые показатели:

1. `buildingAreaM2`
2. `roofAreaM2`
3. `wallAreaM2`
4. `metalMassTons`
5. `hasSnowGuards`
6. `hasRoofFence`
7. `drainageLengthM`
8. `regionMarkupKey`

### 4. UI-слой

Лучше не смешивать это с текущими табами подбора профилей.

Рекомендуемый вариант:

1. новая вкладка `СМР`
2. внутри:
   - краткая сводка базы расчёта
   - стоимость по разделам
   - итоговая стоимость
   - применённый региональный коэффициент
   - примечание, что расчёт укрупнённый

## Этапы Внедрения

### Этап 1. Подготовка reference-data

Сделать extractor матрицы в generated TypeScript-модуль.

Результат:

- матрица читается из Excel автоматически
- нет ручного копирования цен

### Этап 2. MVP по металлоконструкциям

Сделать только:

- стоимость монтажа металлокаркаса

Вход:

- суммарная масса металлоконструкций
- тип каркаса
- регион
- ценовой класс/уровень

Выход:

- стоимость монтажа металлоконструкций

### Этап 3. Коробка здания

Добавить:

- земляные работы
- бетон
- полы
- стены
- кровлю

### Этап 4. Расширенные разделы

Добавить:

- двери / окна / ворота
- инженерные сети
- дополнительные работы

### Этап 5. Печатная форма / коммерческое КП

Добавить:

- печатный лист СМР
- экспорт PDF
- укрупнённую смету по разделам

## Что Можно Сделать Без Новых Полей Вообще

Прямо сейчас без расширения input можно сделать только очень укрупнённый расчёт:

1. монтаж металлоконструкций
2. часть коробки по площади здания
3. часть кровли по площади кровли

Но такой расчёт будет с большим количеством допущений.

2. Сделать extractor матрицы

4. Добавить недостающие поля для коробки
5. Добавить вкладку `СМР`
6. Добавить печатную форму

## Мой Практический Вывод

Лучший следующий шаг:

1. не пытаться сразу автоматизировать весь файл
2. начать с раздела `Металлоконструкции СМР`
3. после этого расшириться до `коробки здания`

Именно так можно получить полезный результат быстро и без сильного риска ошибочной “полной сметы”.
