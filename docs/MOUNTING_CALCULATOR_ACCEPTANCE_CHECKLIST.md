# Чек-Лист Приёмки Работы По Калькулятору Монтажа

## Совместимость

1. Использован тот же стек, что в `metalcalc`
2. Использован `TypeScript`
3. Использован `Vite`
4. Использован `Zod`
5. Использованы `Vitest` и `Playwright`
6. Архитектура совместима с `domain / pages / shared / scripts`

## Контракты И Имена Полей

1. Общие поля не переименованы
2. Поля `city`, `roofType`, `spanM`, `buildingLengthM`, `buildingHeightM`, `frameStepM`, `terrainType` совпадают по названию
3. Экономические поля совпадают по названию
4. Новые поля добавлены поверх существующих, а не вместо них
5. Нет дублирующих полей с другим именем при том же смысле

## Excel / Reference Data

1. Расценки читаются из Excel-матрицы
2. Есть отдельный extractor-скрипт
3. Есть `*.generated.ts`
4. Нет ручного копирования цен в UI-код
5. Источник данных документирован

## Расчёт

1. Реализован MVP по `Металлоконструкции СМР`
2. Расчёт работает от массы конструкций
3. Есть явное правило выбора ценовой строки
4. Это правило описано в документации
5. Итоговая стоимость считается детерминированно
6. Региональный коэффициент учитывается явно

## Тесты

1. Есть unit-тесты на input/schema
2. Есть unit-тесты на расчёт
3. Есть parity-smoke тесты против Excel
4. Есть e2e smoke на UI
5. Все тесты проходят

## UI

1. Есть экран или вкладка расчёта монтажа
2. Есть блок с общей сводкой
3. Есть таблица стоимости по разделам
4. Есть общий итог
5. Есть примечание, что расчёт укрупнённый
6. UI визуально совместим с текущим приложением

## Документация

1. Описано, что считается
2. Описано, что не считается
3. Описаны допущения
4. Описаны новые поля
5. Описаны правила выбора строк матрицы

## Критерий Приёмки

Работа принимается только если:

1. модуль совместим по стеку
2. модуль совместим по контрактам
3. reference-data берутся из Excel
4. расчёт покрыт тестами
5. модуль можно встроить в `metalcalc` без большого рефакторинга
